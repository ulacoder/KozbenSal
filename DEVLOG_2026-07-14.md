## What I Did Today

### 1. Fixed Camera Integration Issues
**Problem:** App wasn't capturing video from PS3 Eye camera

- Added DirectShow backend (`cv2.CAP_DSHOW`) for Windows compatibility
- Created camera test script to verify all camera devices

**Result:** ✅ Camera now captures reliably at 640x480@30fps

---

### 2. Redesigned Drawing Interaction: Dwell → Blink-Based
**Previous system:** User had to hold gaze for 1.5 seconds (dwell time) to activate drawing
- Slow, unintuitive
- Hard to control start/stop
- Continuous stroke drawing

**New system:** Point-to-point drawing with blink activation
- **Blink once** → place first point (green marker)
- **Move gaze** → see preview line
- **Blink again** → draw line between points

**Implementation:**
- Added Eye Aspect Ratio (EAR) calculation for blink detection
- Threshold: EAR < 0.21 for 2 consecutive frames = blink
- Changed `DrawMode` enum: `NONE → WAITING → NONE` cycle
- Visual feedback: blue cursor (waiting point 1) → orange cursor (waiting point 2)

**Result:** ✅ Much faster and more precise drawing control

---

### 3. Improved Pupil Detection with Computer Vision
**Problem:** MediaPipe iris tracking alone wasn't accurate enough for precise gaze control

**Solution:** Hybrid approach combining MediaPipe + traditional CV
- Extract eye region from MediaPipe landmarks
- Apply Gaussian blur (7x7 kernel) to reduce noise
- Threshold dark regions (pupil is darkest part of eye)
- Find largest contour → that's the pupil
- Fall back to MediaPipe iris if CV fails

**Code additions:**
- `_detect_pupil_in_region()` - CV-based pupil finder
- Hybrid logic: prefer CV pupil, fallback to MediaPipe
- Debug visualization shows which method is active

**Result:** ✅ More accurate gaze point, especially in good lighting

## Key Learnings

1. **DirectShow matters on Windows** - Default OpenCV backend failed, CAP_DSHOW fixed it
2. **Hybrid > Pure ML** - Combining classic CV with ML gives better results than either alone
3. **Blink detection is tricky** - Need to tune EAR threshold and consecutive frame count to avoid false positives
4. **User feedback is critical** - Can't know if interaction works until real users try it
